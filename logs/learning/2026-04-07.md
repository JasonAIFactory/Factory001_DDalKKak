#### Render + Supabase 연결 실패 — 3가지 원인
1. **`?pgbouncer=true` 파라미터**: asyncpg 드라이버가 이 파라미터를 인식 못 함 → `TypeError: connect() got an unexpected keyword argument 'pgbouncer'`
2. **Pooler URL vs Direct URL**: Supabase `pooler.supabase.com:6543`은 PgBouncer 프록시. asyncpg는 Direct URL `db.xxx.supabase.co:5432` 사용해야 함
3. **Render Free → Supabase 네트워크**: `OSError: [Errno 101] Network is unreachable` — Render Free 플랜에서 외부 DB 연결 제한 가능성

**해결:** Render 자체 PostgreSQL (Internal) 사용 → 같은 네트워크라 연결 문제 없음

#### DATABASE_URL 형식 — 드라이버 지정 필수
```
SQLAlchemy가 URL 앞부분으로 드라이버 결정:
  postgresql://           → psycopg2 (동기) — 우리 코드와 호환 안 됨
  postgresql+asyncpg://   → asyncpg (비동기) — 우리 코드가 이걸 씀

Supabase/Render가 주는 URL: postgresql://...
우리가 써야 하는 URL: postgresql+asyncpg://...
→ 앞에 +asyncpg만 붙이면 됨
```

#### Internal vs External Database URL (Render)
```
Internal: dpg-xxx-a                    ← 같은 Render 네트워크 안에서만 접근
External: dpg-xxx-a.xxx.render.com     ← 외부에서 접근 (느림, 보안 약함)

백엔드와 DB가 같은 Render에 있으면 → Internal 사용 (빠르고 안전)
```

#### `import os` 누락 — 배포 시 발견된 버그
- 로컬 Docker에서는 문제 없었는데 Render 배포 시 `NameError: name 'os' is not defined`
- `main.py`의 lifespan 함수에서 `os.environ.get("ENVIRONMENT")` 호출
- 원인: import 누락
- 교훈: **모든 파일에서 사용하는 모듈은 명시적 import 필수**

#### Next.js Route Group `(marketing)` — 로컬 vs 프로덕션 차이
```
구조:
  app/(marketing)/page.tsx   ← Route Group (괄호 = URL에 안 나타남)
  app/page.tsx               ← 삭제됐었음

로컬 dev 서버:
  (marketing)/page.tsx → "/" 로 자동 매핑 ✅

Vercel production build:
  app/page.tsx 없으면 → 404 ❌

해결: app/page.tsx 복원, (marketing)/page.tsx를 re-export
  export { default } from "./(marketing)/page";

교훈: Next.js Route Group은 dev에서 되지만 production에서 안 될 수 있음
      루트 page.tsx는 항상 명시적으로 있어야 안전
```

#### 배포 아키텍처 최종 구성
```
Vercel (무료)              → 프론트엔드 (Next.js)
Render Free                → 백엔드 (FastAPI + Docker)
Render PostgreSQL Free     → DB (같은 네트워크, Internal URL)
Cloudflare                 → 도메인 DNS (daeseon.ai)

환경변수:
  Vercel:  NEXT_PUBLIC_API_URL=https://factory001-ddalkkak.onrender.com
  Render:  DATABASE_URL=postgresql+asyncpg://...@dpg-xxx-a/ddalkkak_db
           SECRET_KEY=xxx
           ENVIRONMENT=production
           CORS_ORIGINS=https://ddalkkak.daeseon.ai

DNS (Cloudflare):
  CNAME ddalkkak     → cname.vercel-dns.com
  CNAME api-ddalkkak → factory001-ddalkkak.onrender.com
```

#### Cloudflare Proxy OFF 이유
```
Proxy ON:  유저 → Cloudflare SSL → Vercel/Render SSL → 이중 암호화 → 에러 가능
Proxy OFF: 유저 → Vercel/Render SSL → 정상

Vercel과 Render가 자체 SSL 있어서 Cloudflare Proxy 불필요
나중에 DDoS 보호 필요하면 ON으로 변경 가능
```
