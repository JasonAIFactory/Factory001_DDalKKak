#### Why $1.73 — API Cost Explosion

#### Preview/Test System — Docker-in-Docker Architecture

**전체 흐름:**
```
유저가 "Test Run" 버튼 클릭
  → POST /api/sessions/{id}/preview
  → 백엔드 preview.py 실행
  → _detect_startup_type() — package.json 있으면 Node.js, main.py 있으면 FastAPI
  → _find_free_port() — socket.bind(('', 0))로 충돌 없는 포트 할당
  → _create_preview_db_schema() — PostgreSQL에 격리된 스키마 생성
  → docker run --detach \
      --publish {랜덤포트}:{앱포트} \
      --volume {worktree}:/app \
      --network factory001_ddalkkak_default \
      {이미지} sh -c "{시작 명령어}"
  → _wait_until_healthy() — /health 응답 올 때까지 폴링
  → 프리뷰 URL을 DB에 저장 + WebSocket으로 브로드캐스트
  → 세션 카드에 "Open App" 링크 표시
  → 유저 클릭 → 새 탭에서 실제 앱 열림
```

**핵심 개념:**
- Docker-in-Docker: API 컨테이너 안에서 호스트 Docker를 제어
- Docker 소켓 마운트: `/var/run/docker.sock`을 API 컨테이너에 마운트
- 이렇게 하면 API가 `docker run` 명령으로 프리뷰 컨테이너를 생성/삭제 가능
- 각 프리뷰 컨테이너는 독립된 포트 + 독립된 DB 스키마 = 완전 격리

**앱 타입별 처리:**
| 앱 타입 | 감지 조건 | Docker 이미지 | 시작 명령 | 컨테이너 포트 |
|---------|-----------|--------------|-----------|-------------|
| FastAPI | main.py 존재 | python:3.11-slim | uvicorn main:app | 8000 |
| Node.js | package.json 존재 | node:20-slim | npm start | 3000 |
| Fullstack | 둘 다 존재 | node:20-slim | npm start | 3000 |

**포트 충돌 방지:**
```python
def _find_free_port() -> int:
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.bind(("", 0))  # OS가 빈 포트 자동 할당
        return s.getsockname()[1]
```
- 여러 세션 동시에 Test Run 해도 각각 다른 포트 할당 = 충돌 없음

**DB 격리 (PostgreSQL Schema):**
- 별도 DB 인스턴스 안 띄움 — 기존 PostgreSQL에 스키마만 추가
- `CREATE SCHEMA IF NOT EXISTS "preview_d45569e953bb"`
- 각 프리뷰는 자기만의 스키마에서 테이블 생성 → 데이터 충돌 없음
- 스키마 생성은 즉시 완료 (DB 서버 추가 기동 필요 없음)

**Docker 소켓이란?**
- `/var/run/docker.sock` = Docker 데몬과 통신하는 Unix 소켓
- 이걸 컨테이너에 마운트하면 컨테이너 안에서 `docker` 명령 실행 가능
- 보안 주의: 소켓 접근 = 호스트 Docker 전체 제어 가능 (프로덕션에서는 제한 필요)

#### Stale Closure Bug (React useCallback)
- 문제: 세션 상세에서 뒤로가기 하면 → 그리드 보이다가 → 자동으로 다시 상세로 돌아감
- 원인: `load()` 함수가 `useCallback([id, detailSession])`으로 생성 → auto-poll의 `setInterval`이 옛날 `load` 참조 → 옛날 `detailSession`이 살아있어서 `setDetailSession(updated)` 호출
- 수정: `useRef`로 현재 `detailSession` 추적, `load()`에서는 ref.current 사용
- 교훈: **React에서 setInterval + useCallback = stale closure 위험. ref로 해결**
