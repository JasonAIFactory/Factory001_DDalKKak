#### Ontology — Separate from Agent Levels
- Agent = "what to do" (action subject)
- Ontology = "how to understand the world" (knowledge structure)
- Without ontology: agent does text matching only
- With ontology: agent has deep domain reasoning
- Construction: LLM extracts entities/relations from documents → Neo4j Knowledge Graph
- Example: "CNC-001 alarm" → bearing wear → product A delivery impact → inspection recommendation
- Three approaches: manual (accurate but slow), LLM auto-extract (fast but needs validation), hybrid (practical best)

#### Single Agent vs Parallel Agents — When to Use Which
- Single agent: full context awareness, zero conflicts, consistent style, but sequential (slow), context window limit
- Parallel agents: 4-5x faster, independent progress, smaller context = more accurate, but no shared knowledge, merge conflicts possible
- Rule of thumb: 1 bug fix → single agent. 4+ modules → parallel agents.
- Optimal: 4-5 parallel agents. Above 5, coordination overhead > speed gains.

#### Parallel Agents Have Fewer Errors Than Single Agent
- Single agent (30 iterations, big task): context fills up → forgets earlier work → cross-file conflicts → high error rate
- Parallel agents (5-8 iterations each, small task): small focused context → only knows its module → no cross-contamination → low error rate
- Evidence: single-agent session produced 13 bugs, Test button failed 8 times, page.tsx grew to 900+ lines
- Parallel would have isolated each module's bugs to its own agent

#### Context Window Optimization Strategy
- Claude Code context = ~200K tokens total
- Effective focus range = ~50K tokens (quality degrades beyond this)
- Layer 1 — Permanent context (auto-loaded every session):
  - CLAUDE.md ~3K tokens (rules, stack, conventions)
  - Memory files ~2K tokens (user info, feedback)
  - Always consumes ~5K → 195K remaining
- Layer 2 — Working context (accumulates during session):
  - File reads ~1K/file, code edits ~0.5K/edit, messages ~0.3K/message
  - 100 messages ≈ 30K tokens
- Layer 3 — Compression:
  - /compact when context hits ~50% (100K)
  - Compresses to ~10K (keeps essentials)
  - CLAUDE.md + Memory restore rules automatically
- Parallel agent context advantage:
  - Single agent: 86K+ tokens (whole project) → focus degrades
  - Each parallel agent: 13-20K tokens (one module only) → maximum focus
- Practical rules:
  - 50+ messages → /compact
  - 10+ files read → /compact
  - Topic change → new session
  - 4+ modules → parallel agents
  - 1 bug fix → single agent
  - Claude seems to forget rules → /compact (reloads CLAUDE.md)
