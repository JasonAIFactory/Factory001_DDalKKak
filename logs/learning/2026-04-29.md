**Concept: how to talk about an AI-assisted solo project at Amazon.**

Three rules I'd codify out of this:

- **"I built it solo with Claude Code"** is the opening, not a footnote.
  Amazon's rubric explicitly evaluates GenAI integration. Saying it up front
  matches their signal.
- **The hardest stories are about AI failures, not AI wins.** The preview
  detection bug — "I treated AI output like human output and got burned" —
  is more interesting than any architecture decision, because it shows the
  candidate can evaluate and refine GenAI output (verbatim from the rubric).
- **The cost router answers the meta-question by design.** It's a system
  that decides whether GenAI is the right tool for each request — not just
  a hand-waved opinion. Concrete > philosophical.

**Honest framing rules I'll keep using:**

| Phrase | Use when |
|---|---|
| "running today" | Code that boots end-to-end and you can demo |
| "designed, not built" | Specs in `docs/` with no implementation |
| "I learned this from a bug" | Lead with failure, not solution |
| "I" not "we" | Solo project — never inflate |
| "runs end-to-end on Railway" | Don't say "production scale" |
