#### Claude.ai Max vs API — completely separate billing
- Claude.ai Max subscription ($100/mo) = web chat only
- Anthropic API = separate pay-per-token billing via console.anthropic.com
- No way to use Max subscription for API calls — intentionally separated by Anthropic
- Same applies to ChatGPT Plus vs OpenAI API

#### BYOK (Bring Your Own Key) model
- Users provide their own API key → costs billed directly to their account
- Platform doesn't pay AI costs — user pays Anthropic directly
- Implementation: store user's API key in DB, pass to executor per session

#### Web Terminal + Claude Code = zero API cost for developers
- Embed xterm.js web terminal → user runs `claude` CLI inside Docker container
- User authenticates with their own Anthropic account (OAuth)
- Max plan includes Claude Code usage → no additional API token cost
- Two product modes: automated AI sessions (API cost) + web terminal (user's plan)

#### Docker container git installation
- Python 3.11-slim doesn't include git by default
- Must add `RUN apt-get update && apt-get install -y git` in Dockerfile
- Without git → FileNotFoundError when creating worktrees
- Fixed: rebuilt image with `docker-compose up -d --build`

#### DalkkakAI vs Claude Code — same core, different packaging
- Both use Claude API + tool use in agentic loop
- Claude Code: runs on user's PC, CLI access, single session
- DalkkakAI: runs in Docker on server, web UI, parallel sessions, auto-deploy
- Core pattern identical: send request → tool calls → execute → loop

#### Claude Code key features Jason should know
- `/compact` — manually compress context when conversation gets long
- `/cost` — check current conversation's API cost
- `/help` — full command list
- `Shift+Tab` — multi-line input mode
- `@filename` — include a specific file in context
- CLAUDE.md is the most important feature — already well-configured
- When context auto-compresses, rules from CLAUDE.md can get dropped → use /compact proactively

#### Claude Code hooks — automated session start checks
- Hooks = shell scripts that run automatically on specific events
- Can trigger on: session start, before tool call, after tool call
- Use case: auto-check if today's log files exist, create if missing
- Prevents human error (Claude forgetting CLAUDE.md rules after context compression)
- Configuration lives in Claude Code settings.json

#### Pydantic + Anthropic SDK compatibility
- anthropic SDK (latest) requires pydantic >= 2.7.0
- pydantic 2.6.1 has `by_alias: NoneType cannot be converted to PyBool` bug
- Fix: upgrade pydantic to >= 2.7.0 in requirements.txt
