#### Single Agent vs Parallel Agent — 실전 비교
- **Single Agent (큰 작업)**: context window 꽉 참 → 앞 작업 잊음 → 실수 증가
- **Parallel Agent (작은 작업 여러 개)**: 각자 context 작음 → 집중 → 실수 감소
- page.tsx 900줄을 단일 agent로 유지 → 13개 버그
- 9개 컴포넌트로 분리 후 병렬 → 빌드 에러 0개
- **결론: 작은 context = 더 정확한 결과**

#### Context Window 최적화 전략
- Claude Code context ~200K 토큰, 실제 집중력 유지 ~50K
- Layer 1 (영구): CLAUDE.md 3K + Memory 2K = 항상 5K 소비
- Layer 2 (누적): 파일 읽기 + 수정 + 대화 → 100개 메시지 ≈ 30K
- /compact 시점: 50% (100K) 찼을 때
- 병렬 Agent: 각 15~20K context → 집중력 최대

#### Preview/Test System — Smart App Detection
- 문제: main.py만 하드코딩 → app.py, server.py 못 찾음 → 크래시
- 해결: 우선순위 기반 자동 감지 (룰 기반 → 스마트 감지)
- 순서: dalkkak.json > Procfile > Dockerfile > package.json > *.py 스캔 > index.html
- Python 감지: *.py 파일 내용 스캔 → Flask()/FastAPI()/app.run() import 찾기
- Node.js: package.json의 scripts.start 읽기

#### Agent Level 체계 (1~11)
- Level 1: Single Agent (LLM + Tools)
- Level 4: Meta-Agent (Agent가 다른 Agent 생성/관리)
- Level 5: Self-Improving Agent (자기 프롬프트 수정)
- Level 7: Autonomous Goal Agent (목표만 주면 알아서 실행)
- Level 10: Collaborative Human-AI (사람+AI 동등 협업)
- Level 11: Agent Society (수백 Agent 자율 조직) → 조율 비용 O(n²) 문제
- DalkkakAI 현재: Level 7 + 10 조합
