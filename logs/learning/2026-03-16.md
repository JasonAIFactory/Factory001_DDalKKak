#### Monolith-first architecture — 2026-03-16
What: Ship one process handling everything, extract services only when you can measure the bottleneck
Why: Solo founders waste months on premature microservices. Monolith → measure → extract the hot path
Use: "We started monolithic for velocity, then extracted the hot path into Go microservices as we scaled"

#### Strangler fig pattern — 2026-03-16
What: Gradually replace monolith pieces with microservices, one by one, without rewriting everything
Why: DalkkakAI's Phase 1→2 migration follows this — replace API gateway with Go first, then WebSocket, then session manager
Use: "We're using the strangler fig pattern to incrementally migrate high-throughput services from Python to Go"

#### AI cost router pattern — 2026-03-16
What: Classify every request before selecting an AI model — zero-cost first, then Haiku, Sonnet, Opus
Why: 60% of actions need zero AI. Routing saves 80% on AI costs
Use: "We implemented tiered model routing that reduced AI costs by 80% through intelligent request classification"

#### Spec-driven development — 2026-03-16

Why: Both AI agents and human teams need a shared contract to produce compatible code
Use: "We use spec-driven development where SPEC.md serves as the single source of truth"

#### SQLAlchemy async session lifecycle — 2026-03-16
What: AsyncSession must be committed/rolled back explicitly — use a context manager dependency that commits on success, rolls back on exception
Why: Forgetting rollback on error leaves DB in inconsistent state; `expire_on_commit=False` keeps objects usable after commit
Use: "Our get_db dependency wraps every request in a try/commit/except rollback block"

#### Soft delete pattern — 2026-03-16
What: Never DROP rows — add a deleted_at column and filter it out in every query
Why: Audit trail, data recovery, GDPR compliance — you can always restore soft-deleted records
Use: "We use soft deletes across all tables so nothing is ever permanently lost"

#### Git worktree for parallel AI sessions — 2026-03-16
What: `git worktree add <path> <branch>` creates a second checkout of the same repo on a different branch
Why: Each AI session needs isolated files — two agents can edit different features simultaneously without conflicts
Use: "Each session runs in its own worktree, so 5 parallel agents never touch each other's files"

#### passlib + bcrypt 5.x incompatibility — 2026-03-16
What: passlib 1.7.4 accesses `bcrypt.__about__.__version__` which was removed in bcrypt 4+
Why: Using `passlib[bcrypt]` with bcrypt 5.0.0 causes `AttributeError` → `ValueError` in hash_password
Fix: Remove passlib, use `bcrypt` directly: `bcrypt.hashpw(plain.encode(), bcrypt.gensalt()).decode()`
Lesson: Always use libraries directly when only using 1-2 functions. Wrappers have their own compatibility lifecycle.

#### SQLAlchemy MissingGreenlet after flush — 2026-03-16
What: After `db.flush()`, SQLAlchemy expires instance attributes. Pydantic's `model_validate()` accesses them synchronously → triggers lazy async I/O → MissingGreenlet error
Why: SQLAlchemy async sessions expire attributes after flush to force re-read from DB. But outside an async context (inside Pydantic), the reload fails.
Fix: Add `await db.refresh(obj)` right before `model_validate(obj)` to reload all attributes in async context
Lesson: Any time you flush and then serialize with Pydantic, refresh the object first.

#### SQLite UUID column with AsyncSession — 2026-03-16
What: SQLAlchemy's UUID column type calls `.hex` on the value before sending to SQLite. Strings don't have `.hex` — UUID objects do.
Why: PostgreSQL coerces string → UUID automatically. SQLite does not.
Fix: Convert string to `uuid.UUID(user_id)` before using in `WHERE` clause
Lesson: Always pass typed Python objects to SQLAlchemy UUID columns. Don't assume string coercion.

#### Module-based session splitting (SESSION_RULES.md) — 2026-03-16
What: Sessions must own entire directories, not individual files. Never two sessions touching same file.
Why: File-level parallelism causes git merge conflicts. Directory-level parallelism → zero conflicts.
Pattern: Auth module owns backend/auth/* + backend/models/user.py. Projects module owns backend/projects/* etc.
Rule: Define interfaces (in SPEC.md) before sessions start. Shared files are pre-created by orchestrator, sessions only ADD lines.

#### SQLite test isolation: StaticPool is mandatory — 2026-03-16
What: Without StaticPool, SQLAlchemy's connection pool reuses the same in-memory SQLite connection across test functions
Why: Reused connection = same DB shared between tests = data from test A leaks into test B → 409 Conflict on register
Fix: `from sqlalchemy.pool import StaticPool` + `poolclass=StaticPool` in create_async_engine
Lesson: In-memory SQLite tests always need StaticPool. This is non-negotiable. Missing it causes silent, confusing failures.

#### Docker Desktop must be running before docker-compose — 2026-03-16
What: `docker-compose up` fails with `open //./pipe/dockerDesktopLinuxEngine: The system cannot find the file specified`
Why: This error means the Docker engine is completely offline — not a config issue, not a port issue
Fix: Start Docker Desktop → wait for whale icon in system tray → then run docker-compose up
Lesson: Always check Docker Desktop is running first when docker-compose fails with pipe errors.
